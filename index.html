xcode-select --install

Go环境搭建

发表于 2016-05-02   |   分类于 Go

安装Go

安装Go开发包

brew install go

设置环境变量

vi ~/.bash_profile

### 打开编辑.bash_profile文件

export GOPATH=$HOME/go

export PATH=$HOME/bin:$GOPATH/bin:$PATH

### 添加环境变量

source ~/.bash_profile

### 保存并生效

测试运行

package main

import "fmt"

func main() {

    fmt.Printf("Hello, world")

}

上述代码保存成一个test.go文件，输入命令go run test.go来测试运行。

输出结果：

Hello, world

配置IDE

安装GoSublime

通过Package Control安装GoSublime

设置环境变量

Sublime——》Preferences——》Package Settings——》GoSublime——》Settings - Default

打开文件后将env改为如下内容

"env": { "GOPATH":"$HOME/go","PATH": "$HOME/bin:$GOPATH/bin:$PATH" },

测试效果

Ctrl+B快捷键，输入: 

go run test.go

希望得到的结果是输出Hello, world，但是报错

Abort. Cannot start MarGo: Exception: Cannot find command `C:\Users\xxx\AppData\Roaming\Sublime Text 3\Packages\User\GoSublime\windows-x64\bin\gosublime.margo_r16.05.01-1_go1.6.1.exe`

运行以下命令安装margo来修正：

go get -u github.com/slene/margo

margo被放置在目录：

GOPATH/bin/

Hexo常用指令

发表于 2016-05-02   |   分类于 博客运营

指令

init

$ hexo init [folder]

新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

server

$ hexo server

启动服务器。默认情况下，访问网址为：http://localhost:4000/。

选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

New

$ hexo new [layout] <title>

新建一篇文章。如果没有设置 layout 的话，默认使用_config.yml中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

generate(或g)

$ hexo generate` 或 `$ hexo g

生成静态文件。

选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 文件生成后立即部署网站

deploy(或d)

$ hexo deploy

部署网站。

选项 描述
-g, –generate 部署之前预先生成静态文件

clean

$ hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

Hexo+Github搭建静态博客

发表于 2016-04-22   |   分类于 博客运营

安装node.js

1.安装Homebrew

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"`

HomeBrew 安装命令地址

2.安装node.js

brew install node

安装hexo

1.安装hexo

npm install hexo-cli -g

2.安装hexo-developer-git

npm install hexo-deployer-git --save

3.创建blog目录

4.初始化hexo

cd到blog目录后执行

hexo init blog
npm install

5.启动hexo服务

hexo s

6.本地预览

http://localhost:4000

关联github

1.在github项目中找到https地址

https://github.com/xxx/xxx.github.io.git

2.修改blog目录中的_config.yml

deploy:
    type: git
    repository: https://github.com/xxx/xxx.github.io.git
    branch: master

冒号后要有一个空格

3.生成ssh key

进入~/.ssh/目录

执行命令

ssh-keygen -t rsa -C "test@qq.com"`

生成id_rsa.pub文件

4.在github填入ssh key信息

Github账户 –> Settings –> SSH keys –> add SSH key

将id_rsa.pub文件中的内容复制并保存

发布文章

1.生成文章

hexo new “postName”
名为postName.md的文件会建在目录/blog/source/_posts下

2.clean

cd到blog目录后执行

hexo clean

3.生成

cd到blog目录后执行

hexo g

4.发布

cd到blog目录后执行

hexo d

安装theme

1.下载theme

cd到 blog 目录下执行如下命令：

git clone https://github.com/iissnan/hexo-theme-next themes/next

2.配置theme

将blog目录下_config.yml里theme的名称landscape修改为next

3.更改theme内容

比如修改名称，描述，头像。
修改blog/_config.yml文件和blog/themes/next/_config.yml文件中对应的属性名称即可。

NexT 使用文档

绑定域名

1.添加CNAME文件

在/blog/themes/landscape/source目录下新建文件名为：CNAME文件，直接将域名写入。

2.万网解析域名

记录类型：CNAME

记录值：xxx.github.io.（不要忘了最后的”.”号）
